📖 知识库使用指南

基于 Knowledge-Base 协议 + bot-url-question-router 行为规则
服务器：京东云 | 工作空间：/root/notes/

一、知识库结构

二、两种使用方式

方式 A：通过 959972276-normal 机器人（自动）

这是最常用的方式，给 QQ 机器人发消息即可：

📎 收到 URL → 自动抓取入库

步骤 说明
① 抓取 先用 web_fetch，失败则用 Web Content Fetcher（r.jina.ai → markdown.new → defuddle.md）
② 提取 标题、来源、日期、作者、摘要、正文
③ 入库 写入 Notes/02-Sources/SRC-xxx-YYYY-MM-DD.md + Fulltext/
④ 回复 📥 已收录 → 知识库

支持链接类型： 文章、博客、推文、CSDN、PDF（尝试抓取）、短链接（自动跟踪重定向）

❓ 收到问题 → 语义检索知识库

步骤 说明
① 搜索 语义检索 /root/notes/ 全部笔记
② 匹配 返回 1-3 条最相关结果 + 来源路径
③ 无匹配 直接告知”没找到”

问候语（hi/你好/在吗）不走检索。

方式 B：直接对 AI 说（手动）

在 WebChat 或任何对话中，直接用自然语言告诉我做什么：

你说的话 我会做什么
“记个笔记：xxx” 判断类型 → 写入对应目录
“写日报：今天搞定了xxx” 新建 01-Daily/DAILY-YYYY-MM-DD.md
“帮我存一下这篇文章：[URL]” 抓取 → 入库 02-Sources/
“查一下之前关于xxx的笔记” 检索知识库 → 返回结果
“这周周报：xxx” 新建 08-Reviews/ 周报
“记个故障：MySQL 连不上了” 新建 04-Issues/ISSUE-MySQL-xxx.md
“新建项目：xxx” 新建 05-Projects/PRJ-xxx/
“知识库体检一下” 检查结构完整性、链接健康、命名规范
“归档这个笔记” 移动到 99-Archive/ + 更新链接

三、命名规则速查

每条笔记的文件名必须按规则命名：

笔记类型 文件名格式 示例
日报 DAILY-YYYY-MM-DD.md DAILY-2026-06-30.md
来源资料 SRC-Topic-YYYY-MM-DD.md SRC-K8s-Troubleshooting-2026-06-30.md
全文存档 SRC-Topic-Fulltext-YYYY-MM-DD.md SRC-K8s-Troubleshooting-Fulltext-2026-06-30.md
概念知识 CONCEPT-Domain-Topic.md CONCEPT-Linux-Network.md
问题排查 ISSUE-System-Problem.md ISSUE-Nginx-502.md
项目 PRJ-Project-Name/ PRJ-Blog/
架构 ARCH-Topic.md ARCH-Database-Sharding.md
方法 PATTERN-Scenario.md PATTERN-Deploy-Rollback.md
周报 YYYY-Www-WeeklyReview-MMDD-MMDD.md 2026-W27-WeeklyReview-0629-0705.md
收集箱 INBOX-YYYYMMDD-Title.md INBOX-20260630-Links.md
索引 INDEX-Domain.md INDEX-Server-Overview.md
操作日志 ops-YYYY-MM-DD.md ops-2026-06-30.md

四、Front Matter 模板

每篇笔记头部必须有 YAML 元数据：

type 可选值： daily | inbox | source | concept | issue | project | architecture | pattern | review | index

status 生命周期：

类型 状态流
Source captured → processed → distilled → archived
Concept draft → stable
Issue open → diagnosed → fixed → verified → archived
Project active → paused → archived
Pattern draft → stable

五、核心规则

✅ 必检 6 项（每次写入后自动检查）

  1. 路径符合类型判定规则
  2. 文件名符合命名约定
  3. Front Matter 完整（id/title/type 必填）
  4. updated 已回写当天日期
  5. 不出现敏感信息（密码、Token、API Key 等）
  6. 无重复内容

📌 其他重要规则

  • 防重复： 建长期笔记前先搜索同名/同主题内容，有则更新不新建
  • 相对链接： 内部链接用 [说明](相对路径)，禁止绝对路径
  • 临时收集 7天清理： Inbox 内容 7 天内必须归档或删除
  • 升格原则： Daily 中出现可复用结论、排障闭环、架构决策时，必须升格为长期笔记
  • 敏感信息： 一律用 <占位符> 替代

六、安装的技能清单

技能 来源 说明
Knowledge-Base ClawHub · otweihan 结构化知识库管理协议
Web Content Fetcher ClawHub · mrtommywu URL 内容抓取（r.jina.ai 等）
bot-url-question-router 自定义 959972276-normal 机器人行为路由

七、知识库体检

定期检查内容（可随时说”体检一下知识库”）：

  • 结构完整性 — 目录是否存在、README 是否更新
  • 链接健康 — 相对链接是否有效
  • 命名规范 — 文件名是否符合规则
  • 孤立页面 — 未关联其他笔记的页面
  • 过期内容 — 是否需要归档
