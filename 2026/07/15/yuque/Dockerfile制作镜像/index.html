构建自定义镜像

docker目前镜像的制作有俩种方法：

  • 基于Docker Commit制作镜像
  • 基于dockerfile制作镜像，Dockerfile方式为主流的制作镜像方式

Commit构建镜像

使用 commit 构建自定义镜像：就是将 docker 容器构建成 docker 镜像。

  • 参数说明
    • -a 标注作者
    • -m 说明注释
  • 示例

dockerfile构建自定义镜像示例

  • 查看：docker images

镜像分层结构剖析

  • 总结：
    • 共享资源
    • 对容器的任何改动都是发生容器层
    • 容器层是可写可读，而镜像层只读
    • 每次执行命令都是从容器层开始，自上而下查找
  • 查看docker镜像构建过程命令

Dockerfile基础指令

  • FROM：基于哪个镜像
  • MAINTAINER：注明作者
  • COPY：复制文件进入镜像（只能用相对路径，不能用绝对路径）
  • ADD：复制文件进入镜像（假如文件是.tar.gz文件会解压）
  • WORKDIR：指定工作目录，假如路径不存在会创建路径
  • ENV：设置环境变量
  • EXPOSE：暴露容器端口
  • RUN：在构建镜像的时候执行，作用于镜像层面
  • ENTRYPOINT：在容器启动的时候执行，作用于容器层，dockerfile里有多条时只允许执行最后一条
  • CMD：在容器启动的时候执行，作用于容器层，dockerfile里有多条时只允许执行最后一条，容器启动后执行默认的命令或者参数，允许被修改

命令格式：

  • shell命令格式：RUN yum install -y net-tools
  • exec命令格式：RUN [ "yum","install" ,"-y" ,"net-tools"]

Dockerfile构建JAVA网站镜像

本地安装jdk验证是否成功

  • 将jdk包上传到服务器，解压到 /usr/local/jdk
  • 本地宿主机配置jdk，vi /etc/profile（在最后一行添加）
  • source /etc/profile
  • 检验：java -version

构建JAVA网站镜像

  • vi dockerfile
  • 构建：docker build -t tomcat:8.5 ./
  • 启动容器
  • 在 /data/images/tomcat/ROOT 目录下，执行 echo 'hello wrold' > index.html
  • 打开浏览器访问虚拟机地址

Dockerfile构建nginx镜像

构建nginx镜像

  • dockerfile
  • 安装nginx的shell脚本
  • 构建：docker build -t mycentos:nginx ./
  • Nginx镜像启动注意
    • 在容器里nginx是以daemon方式启动，退出容器时，nginx程序也会随着停止：/usr/local/nginx/sbin/nginx
    • 使用前台方式永久运行：/usr/local/nginx/sbin/nginx -g "daemon off;"
  • 启动容器

  • 编写redis编译安装shell脚本 redis_install.sh
